Design system decision: Lumos is the agreed framework going forward. All new components, CSS tokens, and utility classes should follow Lumos conventions.
Performance
1
Audit and reduce third-party scripts
Lighthouse identifies render-blocking scripts as the top bottleneck. Run a full script audit — identify any unused tags, old tracking pixels, or duplicated analytics snippets and remove them. This alone can move the performance score by 10–15 points.
2
Optimise images across all pages
Convert images to WebP, enable lazy loading on below-the-fold images, and fix any images served at 2–3× their display size. Images are a major contributor to the 25/100 mobile score.
3
Enable Webflow asset compression and CDN settings
Check that Webflow's built-in minification is on, assets are served from the CDN edge, and no custom code is bypassing it. Quick setting check but high leverage.
CSS cleanup
4
Stop adding new duplicate CSS classes
660 numbered class clones already exist (e.g. button-2, u-gap-2-2) — these stay in place. Going forward, use Lumos class names directly instead of letting Webflow create numbered clones when pasting components.
5
Consolidate 3 competing CSS variable systems
Three separate token sets define the same colour values. Consolidate to the Lumos system and find/replace all references to the deprecated --old-* variables (28 still in use).
6
Build new work in Lumos only — leave existing Finsweet in place
Lumos is the agreed design system going forward. Existing Finsweet/Client-First components and classes stay untouched. All new components, layouts, and utilities should be built using Lumos — no new Finsweet code added to the project.
Components
7
Replace 8 legacy components still live on pages
8 of 11 legacy components are still rendering on live pages. Swap each for its Lumos equivalent. Low risk since Lumos replacements already exist — this is a find-and-replace job in the Webflow designer.
8
Consolidate 7 navigation components down to 2
There are 7 separate nav variants where there should be 2 (default + light start). The extras are duplicates from different agencies each making their own version. Consolidate and replace across all pages.
9
Fix naming on Footer and NavBar components
Footer (primary) and both NavBar variants have mixed naming conventions — some classes use camelCase, some use BEM, some are unlabelled. Low risk cleanup, just rename in the Webflow style manager.
Analytics
10
Install Microsoft Clarity
Currently zero session recording or heatmap visibility. PhotoDay creates a free Clarity account, gets the snippet, and it gets added via Google Tag Manager. Takes about 10 minutes and provides immediate user behaviour data once traffic flows.
