Photoday.com — site repair report

June 2026

A summary of what needs to be fixed on photoday.com, in order of priority.

Critical
Important
Low risk
Quick win
Site speed
1
Find and remove scripts that are slowing the site down
The site loads too many background scripts — old tracking codes, unused tools, and duplicates that nobody cleaned up. We need to go through them, remove the ones that aren't needed, and keep only what's actually in use. This is the single biggest thing we can do to improve load speed.
2
Fix images — they're too heavy and slowing down every page
Many images on the site are much larger than they need to be, which makes pages slow to load — especially on mobile. We'll convert them to a lighter format and make sure images only load when the visitor actually scrolls to them.
3
Turn on Webflow's built-in speed settings
Webflow has built-in options that compress files and serve them faster — they just need to be switched on. A quick settings check with immediate impact on load time.
Styling
4
Stop creating style duplicates when adding new sections
Over time, 660 extra style copies have built up on the site from different people pasting in components. The ideal fix is to remove all unused classes entirely — if that's not possible, the minimum is to stop adding new ones. Going forward, use Lumos class names directly to avoid creating new numbered clones.
5
Use one colour system across the whole site
The site currently has 3 different places where colours are defined, which leads to inconsistencies. Consolidate everything to Lumos tokens — this also means removing all unused colour variables that are no longer referenced anywhere on the site.
6
All new work built with Lumos — nothing existing removed
Going forward, all new sections and components will be built using the Lumos design system. Existing work stays untouched — this is purely about making sure everything added from here on follows one consistent approach.
Page sections
7
Update 8 outdated page sections to the current versions
8 sections across the site are still using old versions that should have been replaced. The updated versions already exist — it's just a case of swapping them in page by page.
8
Clean up the navigation — 7 versions exist, only 2 are needed
Over time, 7 slightly different versions of the navigation have been created by different teams. Only 2 are actually needed. We'll replace the extras with the correct versions across all pages.
9
Tidy up the names on the footer and navigation sections
The footer and navigation have inconsistent internal names, making them harder to work with. A straightforward rename to bring them in line with everything else — no visual changes to the site.
Analytics
10
Set up visitor recordings so we can see how people use the site
Right now there's no way to see where visitors click, where they drop off, or what's confusing them. Microsoft Clarity is a free tool that records sessions and shows heatmaps. Quick to set up — gives immediate insight into how the site is actually being used.
Bigger picture
11
Consider a full rebuild
Probably the easiest way to truly get this site on the right track — lightweight, fast, and built consistently from the ground up — is a full rebuild. Years of different agencies working in their own way has created layers of debt that are difficult to untangle piece by piece. If a rebuild isn't on the table, the fixes above are the next best path forward.
